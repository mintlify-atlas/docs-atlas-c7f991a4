Broń Vault is deployed as a set of Docker services. This guide walks you through every step — from cloning the repository to uploading your first stealer log — using the recommended Docker Compose quick-start path.

<Warning>
  Broń Vault is intended for use in secure, isolated environments only. Do **not** expose it to public networks or deploy it in production. Use it exclusively on a trusted local machine or isolated lab network.
</Warning>

## Prerequisites

Before you begin, ensure you have the following installed:

| Requirement | Notes |
|---|---|
| **Docker** + **Docker Compose v2** | Docker Desktop on macOS/Windows, or `docker-ce` + `docker-compose-plugin` on Linux |
| **Git** | For cloning the repository |

**Tested environments:** Ubuntu 24.04 LTS (Desktop and Server), Kali Linux 2025.3, macOS Sequoia.

On Ubuntu you can install Docker automatically using the bundled script:

```bash
bash install_docker.sh
```

<Note>
  After running `install_docker.sh`, log out and back in so your user is added to the `docker` group before continuing.
</Note>

## Installation

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/ITSEC-Research/bron-vault
    cd bron-vault
    ```
  </Step>

  <Step title="Configure your environment">
    Copy the example environment file and set secure passwords for all services:

    ```bash
    cp .env.example .env
    ```

    Open `.env` and replace the placeholder values:

    | Variable | Description | Example |
    |---|---|---|
    | `JWT_SECRET` | Secret for signing auth tokens. Minimum 32 characters. | `openssl rand -hex 32` |
    | `MYSQL_ROOT_PASSWORD` | MySQL root password | `MySecureRootPass!` |
    | `MYSQL_USER` | Application database user | `bronvault` |
    | `MYSQL_PASSWORD` | Application database password | `MySecureAppPass!` |
    | `CLICKHOUSE_USER` | ClickHouse user | `default` |
    | `CLICKHOUSE_PASSWORD` | ClickHouse password | `MyClickHousePass!` |
    | `CLICKHOUSE_DB` | ClickHouse database name | `bronvault_analytics` |
    | `SYNC_USER` | Internal MySQL→ClickHouse replication user | `clickhouse_sync` |
    | `SYNC_PASSWORD` | Password for replication user | `MySyncPass!` |

    <Warning>
      Change **all** default passwords before starting the services. Never commit your `.env` file to version control.
    </Warning>
  </Step>

  <Step title="Start all services">
    ```bash
    bash docker-start.sh
    ```

    The script will:
    - Build Docker images (first run only — subsequent starts are faster)
    - Start MySQL, ClickHouse, MinIO, and the Next.js application
    - Run the setup container to configure MaterializedMySQL replication and initialize all database tables
    - Print service status and access URLs once everything is ready

    <Note>
      First startup takes approximately 60 seconds while MySQL and ClickHouse initialize and the setup script configures replication.
    </Note>
  </Step>

  <Step title="Open the application">
    Navigate to `http://localhost:3000` in your browser. Log in with the default credentials:

    | Field | Value |
    |---|---|
    | Email | `admin@bronvault.local` |
    | Password | `admin` |

    <Warning>
      Change the default password immediately after first login via **User Settings → Change Password**.
    </Warning>
  </Step>

  <Step title="Upload your first log file">
    Navigate to **Upload** in the sidebar, drag and drop a stealer log `.zip` file onto the upload area, and click **Upload**. Real-time progress is shown as the parser extracts devices, credentials, files, and software from the archive.
  </Step>
</Steps>

## Service URLs

Once all services are running:

| Service | URL | Notes |
|---|---|---|
| Broń Vault App | `http://localhost:3000` | Main web interface |
| ClickHouse Play | `http://localhost:8123/play` | Interactive SQL query UI |
| ClickHouse HTTP API | `http://localhost:8123` | Direct API access |
| MySQL | `localhost:3306` | Primary database |
| MinIO S3 API | `http://localhost:9001` | Object storage endpoint |
| MinIO Console | `http://localhost:9002` | MinIO web management UI |

## Useful commands

```bash
# Check service status
./docker-status.sh

# Stream logs from all services
docker compose logs -f

# Stop all services
docker compose down

# Restart services
docker compose restart
```
