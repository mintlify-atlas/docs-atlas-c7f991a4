---
title: "Search Credentials — POST /api/v1/search/credentials"
description: "Search stealer log credentials by email, username, password, or any field. Supports pagination and optional full-password inclusion."
---

The credentials search endpoint lets you query every credential record in Broń Vault across all uploaded log archives. Results include the matched credential along with device context and host system information extracted from the stealer log. This is the primary endpoint for investigating exposure of a specific email address or username.

- **Method:** `POST`
- **Path:** `/api/v1/search/credentials`
- **Auth:** `admin` or `analyst` API key

<Note>
  Password searches (`"type": "password"`) use exact matching only, not pattern matching, to prevent overly broad exposure of sensitive data.
</Note>

## Request body

<ParamField body="query" type="string" required>
  The search term. Minimum 3 characters, maximum 500 characters. For `email` and `username` types this is a substring match (`ILIKE`). For `password` type it is an exact match.
</ParamField>

<ParamField body="type" type="string" default="any">
  Constrains which field to search. Accepted values:

  - `any` — matches username or URL (default)
  - `email` — matches username fields that contain `@`
  - `username` — matches any username substring
  - `password` — exact password match
</ParamField>

<ParamField body="page" type="number" default="1">
  Page number (1-indexed).
</ParamField>

<ParamField body="limit" type="number" default="50">
  Results per page. Maximum is 100.
</ParamField>

<ParamField body="includePasswords" type="boolean" default="false">
  When `true`, each result includes the full plaintext `password` field in addition to the masked `passwordMasked` field. Omit or set to `false` to receive masked passwords only.
</ParamField>

## Response

<ResponseField name="success" type="boolean" required>
  `true` on a successful response.
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="properties">
    <ResponseField name="credentials" type="object[]">
      Array of matched credential records. See credential object fields below.
    </ResponseField>

    <ResponseField name="pagination" type="object">
      <Expandable title="properties">
        <ResponseField name="page" type="number">Current page number.</ResponseField>
        <ResponseField name="limit" type="number">Results per page.</ResponseField>
        <ResponseField name="total" type="number">Total number of matching records.</ResponseField>
        <ResponseField name="totalPages" type="number">Total number of pages.</ResponseField>
        <ResponseField name="hasMore" type="boolean">Whether additional pages exist.</ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="meta" type="object">
  <Expandable title="properties">
    <ResponseField name="query" type="string">The query string that was searched.</ResponseField>
    <ResponseField name="type" type="string">The search type that was used.</ResponseField>
    <ResponseField name="searchedAt" type="string">ISO 8601 timestamp of when the search ran.</ResponseField>
  </Expandable>
</ResponseField>

### Credential object

<ResponseField name="deviceId" type="string">
  Unique identifier of the device this credential was extracted from.
</ResponseField>

<ResponseField name="deviceName" type="string">
  Human-readable name of the device (computer name from host information, falling back to the device record name).
</ResponseField>

<ResponseField name="url" type="string">
  Full URL where the credential was saved in the browser.
</ResponseField>

<ResponseField name="domain" type="string">
  Extracted domain component of the URL.
</ResponseField>

<ResponseField name="username" type="string">
  Captured username or email address.
</ResponseField>

<ResponseField name="passwordMasked" type="string">
  Partially masked password. The first and last characters are visible; middle characters are replaced with asterisks (e.g., `p******d`).
</ResponseField>

<ResponseField name="password" type="string">
  Full plaintext password. Only present when `includePasswords` is `true` in the request.
</ResponseField>

<ResponseField name="browser" type="string">
  Browser from which the credential was stolen (e.g., `Chrome`, `Firefox`). `Unknown` if not detected.
</ResponseField>

<ResponseField name="country" type="string">
  Two-letter country code of the device, if available.
</ResponseField>

<ResponseField name="uploadDate" type="string">
  Date the containing archive was uploaded to Broń Vault.
</ResponseField>

<ResponseField name="hostInfo" type="object">
  Host system information extracted from the stealer log. All fields are optional and may be `null` if not present in the log.

  <Expandable title="hostInfo properties">
    <ResponseField name="os" type="string">Operating system string.</ResponseField>
    <ResponseField name="ipAddress" type="string">External or local IP address of the victim device.</ResponseField>
    <ResponseField name="machineUsername" type="string">Windows username on the victim machine.</ResponseField>
    <ResponseField name="cpu" type="string">CPU model string.</ResponseField>
    <ResponseField name="ram" type="string">RAM amount string.</ResponseField>
    <ResponseField name="gpu" type="string">GPU model string.</ResponseField>
    <ResponseField name="hwid" type="string">Hardware ID (HWID) of the device.</ResponseField>
    <ResponseField name="antivirus" type="string">Detected antivirus software.</ResponseField>
    <ResponseField name="stealerType" type="string">Stealer family that generated the log (e.g., `Redline`, `Vidar`).</ResponseField>
    <ResponseField name="logDate" type="string">Date the stealer captured the data on the victim device.</ResponseField>
  </Expandable>
</ResponseField>

## Error responses

| Code | HTTP | Description |
|------|------|-------------|
| `INVALID_REQUEST` | 400 | `query` is missing or not a string |
| `QUERY_TOO_SHORT` | 400 | `query` is fewer than 3 characters |
| `QUERY_TOO_LONG` | 400 | `query` exceeds 500 characters |
| `INVALID_API_KEY` | 401 | Key is missing or invalid |
| `SEARCH_ERROR` | 500 | Internal error during query execution |

## Examples

<CodeGroup>

```bash Search by email
curl --request POST \
  --url http://localhost:3000/api/v1/search/credentials \
  --header "Authorization: Bearer bv_YourApiKeyHere" \
  --header "Content-Type: application/json" \
  --data '{
    "query": "alice@example.com",
    "type": "email",
    "page": 1,
    "limit": 10,
    "includePasswords": false
  }'
```

```bash Search by username with passwords
curl --request POST \
  --url http://localhost:3000/api/v1/search/credentials \
  --header "Authorization: Bearer bv_YourApiKeyHere" \
  --header "Content-Type: application/json" \
  --data '{
    "query": "alice",
    "type": "username",
    "includePasswords": true
  }'
```

</CodeGroup>
