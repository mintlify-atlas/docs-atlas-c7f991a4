The credential search page is the primary interface for investigating exposed accounts. Enter one or more email addresses, domains, or usernames and Broń Vault queries ClickHouse across every log you have uploaded. Results include the full credential context and, when a match is found, a **Supporting Files** tab that surfaces every other artefact from the same compromised device.

## Search basics

Type any of the following into the search bar on the home page:

| Input type | Example | What it matches |
|---|---|---|
| Email address | `alice@example.com` | Credentials where the username contains that email |
| Domain | `example.com` | Credentials and URLs targeting that domain or its subdomains |
| Username / partial | `alice` | Credentials where the username contains the term |

Searches are case-insensitive. Domain searches automatically match subdomains: searching `example.com` will also return results for `login.example.com`, `mail.example.com`, and so on.

## Boolean operators

Combine terms with the three Boolean operators to write precise queries:

<CardGroup cols={3}>
  <Card title="OR — comma `,`" icon="code-branch">
    Returns credentials matching **any** of the listed terms.

    ```
    example.com, test.com
    alice@example.com, bob@example.com
    ```
  </Card>
  <Card title="AND — plus `+`" icon="plus">
    Returns devices that have credentials matching **all** listed terms simultaneously.

    ```
    example.com + admin
    alice@corp.com + corp.com
    ```
  </Card>
  <Card title="NOT — minus prefix `-`" icon="minus">
    Excludes credentials matching the prefixed term.

    ```
    example.com, -staging.example.com
    corp.com, -test
    ```
  </Card>
</CardGroup>

Operators can be composed. Commas separate OR groups; `+` creates AND logic within a group:

```
a.com + b.com, c.com
```

This returns devices that have credentials for **both** `a.com` AND `b.com`, **OR** devices that have credentials for `c.com`.

## Advanced syntax

<Accordion title="Wildcard matching with *">
  Place `*` anywhere in a term to match any characters at that position:

  ```
  admin*@gmail.com    → matches admin123@gmail.com, admin_test@gmail.com
  *.example.com       → matches all subdomains
  pass*               → matches any username or domain starting with "pass"
  ```
</Accordion>

<Accordion title='Exact match with "..."'>
  Wrap a term in double quotes to require an exact, case-insensitive match with no substring expansion:

  ```
  "admin@example.com"    → matches only that exact email address
  "example.com"          → matches only that exact domain, not subdomains
  ```
</Accordion>

<Accordion title="Field-prefixed search">
  Target a specific column with a `field:value` prefix:

  | Prefix | Aliases | Column searched |
  |---|---|---|
  | `domain:` | `d:` | Credential domain field |
  | `user:` | `username:`, `email:`, `u:` | Username / email field |
  | `url:` | — | Full URL field |
  | `browser:` | `b:` | Browser name |
  | `password:` | `pass:`, `p:` | Password field |

  **Example:**
  ```
  domain:example.com + user:alice
  browser:Chrome, browser:Edge
  ```
</Accordion>

## Reading search results

Each result row in the table shows:

- **URL** — the full credential URL
- **Domain** — the extracted hostname
- **Username** — the account login (email or username)
- **Password** — masked by default; click the eye icon to reveal
- **Browser** — the browser from which the credential was stolen
- **Country** — the country of the infected device (from system information)
- **Device link** — a direct link to the full [device detail view](/features/device-detail)

Results are paginated and can be sorted by any column.

## Supporting Files tab

When your search returns results, a **Supporting Files** tab appears alongside the main **Credentials** tab. This tab shows all other data extracted from the same device as the matched credential:

<CardGroup cols={2}>
  <Card title="Cookies" icon="cookie">
    Browser cookies from the same device, useful for session analysis.
  </Card>
  <Card title="Browser history" icon="clock-rotate-left">
    Visited URLs recorded by the stealer.
  </Card>
  <Card title="System files" icon="folder-open">
    Other files extracted from the device — configuration files, saved passwords, documents.
  </Card>
  <Card title="Device link" icon="server">
    Quick-access button to open the full device detail page for deeper investigation.
  </Card>
</CardGroup>

## Pivoting to device detail

From any search result, click the device link in the **Device** column (or the link in the Supporting Files tab) to open the [Device Detail view](/features/device-detail). This shows a complete picture of the compromised machine: hardware specs, all credentials, installed software, the full file tree, and extracted host information across all stealer families detected on that device.

<Tip>
  Use the AND operator (`+`) to find devices that have credentials for **two different targets simultaneously** — a strong indicator of high-value machines worth investigating in detail.
</Tip>
